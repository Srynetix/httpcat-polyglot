# httpcat-java - Simple, stupid http.cat client made in Java

Show HTTP status code as cute images of cats (or dogs!), with colors!

## Experience summary

Oh my, the comeback!

You know, Java is not really my thing, and weirdly I'm doing more and more software architecture so I'm using more and more paradigms which are really common in Java.

But the Java ecosystem, it's not for me. Just starting with gradle, which adds some wrappers in your project so you can install Gradle (why?). The Java/Kotlin/Groovy melting-pot also makes everything a little confusing.

Finding curated libs in Java was the hardest part. After one search I ended up on Apache libraries (like the Apache HttpClient libraries), and I was just lost. Things like this make you miss npm/crates.io.

In the end, I found the libs I needed, and was positively surprised by the `picocli` library, nice to use to setup a CLI.

## Packages and versions

- Java 21

- For CLI parsing: `info.picocli:picocli`
- To convert images to ASCII: `io.github.seujorgenochurras:image-to-ascii`

- To test: `junit-jupiter`

## How-tos

### Build

```sh
$ gradle jar
```

$ java -jar ./app/build/libs/app.jar 200
